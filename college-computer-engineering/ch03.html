### Cache Replacement Strategies

#### 1. **FIFO (First In, First Out)** When new data needs to be written into Cache, if there is an empty line, write it into the empty line. Otherwise, replace one line.

#### 2. **LRU (Least Recently Used)** Replace the least recently used line. This method is more reasonable than FIFO.

#### 3. **Random Replacement** Each Cache line has a counter. When replaced, choose the line with the highest counter. When a line is loaded or replaced, its counter is reset to 0. If it is a hit, the counter of the hit line is incremented by 1. If it is a miss, the counter of the miss line is incremented by 10.

### Cache Write Strategies

#### 1. **Write Through** Write into Cache and simultaneously write into main memory. This is consistent with the write policy. When replaced, no write is required; it reduces the write speed.

#### 2. **Write Back (with Buffer)** Write the data into the write buffer first. This does not affect the CPU speed. The write buffer can be written back to memory.

#### 3. **Write Back (dirty)** Write the dirty line into Cache. If the line is replaced, it will not be written back to main memory until the line is replaced. This reduces the write time and ensures Cache consistency.

#### 4. **Write Through (dirty)** Write the dirty line into Cache and invalidate the Cache line. This is not consistent with the write policy and increases the miss rate.

### Memory Management Overview

#### 1. **Process (Task)** A process is a program running on a specific data set. It has independent functions and performs a series of operations.

#### 2. **Logical Address Space** The logical address space is the address space corresponding to the physical address when the program is compiled. It is the address space that the program sees.

#### 3. **Memory Management Unit (MMU)** The MMU is a part of the CPU that converts logical addresses to physical addresses. Segment management uses segment tables, while page management uses page tables. Each segment or page is stored in memory and assigned a segment or page number.

#### 4. **Segment Management** Segments are divided into fixed-length blocks. Each segment has a segment table. The segment table provides the segment's base address in memory, segment number, and segment length.

#### 5. **Page Management** Pages are divided into fixed-length blocks. Each page has a page table. The page table provides the page's base address in memory, page number, and page length.

#### 6. **Fragmentation** Fragmentation occurs when the memory is divided into small blocks, and the memory is not contiguous. This can lead to low efficiency.

#### Section: Cache and Memory Management

When both MMU (Memory Management Unit) and Cache are present, there are two scenarios: 1. **Virtual Address Cache**: Cache is placed before MMU, using virtual addresses. - **CPU → Cache**: If the cache hits, the data is found in the cache. - **Cache Miss**: If the cache misses, the data is fetched from the MMU. - **Advantages**: Cache can be used to improve performance by reducing the number of times the MMU is accessed. - **Disadvantages**: Different virtual addresses may map to the same physical page, causing cache inconsistencies.

2. **Physical Address Cache**: Cache is placed after MMU, using physical addresses. - **CPU → MMU**: If the MMU hits, the data is found in the cache. - **MMU Miss**: If the MMU misses, the data is fetched from the cache. - **Advantages**: Avoids cache inconsistencies. - **Disadvantages**: Cache misses can lead to MMU address translation, which can be time-consuming.

**Disadvantages:** - Clock skew issues. - Sensitive to noise. - DC offset problems.

#### Section 2: DAC0832

**Dual-Buffer Mode:** - Two registers are selected alternately: CS → XFER. (First register AL is used for conversion, second register AL is used for data output.)

**Single-Buffer Mode:** - Single selection: XFER = WR2 = 0; CS. - Data input register: - LE1 = 1, CS = WR1 = 0: Direct LE1 = 1. - WR1 = 1, Hold: LE1 = 0. - DAC register: - XFER = WR2 = 0: Direct LE2 = 1. - WR2 = 1, Hold: LE2 = 0.

#### Section 3: ADC0809

**Channel Selection:** - Select channel using the low three bits of the address. - MOV DX, 220H + n; n = 0~7. - OUT DX, AL; AL value irrelevant. - Read converted data: - MOV DX, 220H; Select ADC, any channel. - IN AL, DX. - Conversion completion signal, high active. - EOC: Conversion completion signal, high active. - Press the interrupt request pin, ADC completion triggers interrupt signal. - Check DB, CPU queries the status of the data register to determine if ADC conversion is complete.
